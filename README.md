# Cylinder + Cube Detection Training Pipeline

Two-class object detection for **cylinders** and **3D-printed PLA cubes**
using **EfficientDet-Lite0**, targeting the **Google Coral USB Accelerator**.

Camera: eye-in-hand on Denso VS-6577E — inference runs once at robot home position.
Output: bounding boxes for cylinders (class 0) and cubes (class 1).

## Workflow

### Step 1 — Annotate with labelme

```bash
labelme dataset/images/ --labels dataset/labels.txt
```

`dataset/labels.txt` must contain:
```
cylinder
cube
```

Rules:
- Background images: no annotation (skip)
- Every visible cylinder and cube in each image must be labelled with the correct class
- Use rectangle bounding boxes only

### Step 2 — Convert labelme → COCO JSON

```bash
/home/nam/.pyenv/versions/gesture_env/bin/python \
  scripts/labelme_to_coco.py \
  --images-dir dataset/images/ \
  --output     dataset/annotations/merged.json
```

Prints annotated image count and per-class box totals — verify both `cylinder`
and `cube` appear before proceeding.

### Step 3 — Train

```bash
/home/nam/.pyenv/versions/gesture_env/bin/python \
  scripts/train.py
```

Pipeline:
1. Load `merged.json`
2. Convert COCO → Pascal VOC XML (`outputs/voc_xml_all/`)
3. Split 70 / 15 / 15 train / val / test into `outputs/voc_split/`
4. Train EfficientDet-Lite0 (50 epochs, batch 8)
5. Evaluate COCO metrics on val + test
6. Export SavedModel, Float32 TFLite, INT8 TFLite
7. Generate outputs (report, plots, sample detections)

### Step 4 — Re-generate Outputs Without Retraining

```bash
/home/nam/.pyenv/versions/gesture_env/bin/python \
  scripts/train.py --skip-training
```

Reloads the existing model and regenerates `training_history.png`,
`detection_report.txt`, and `sample_detections.png`.

### Step 5 — Compile for EdgeTPU

Install the EdgeTPU compiler if not already present:
```bash
curl https://packages.cloud.google.com/apt/doc/apt-key.gpg | sudo apt-key add -
echo "deb https://packages.cloud.google.com/apt coral-edgetpu-stable main" \
  | sudo tee /etc/apt/sources.list.d/coral-edgetpu.list
sudo apt update && sudo apt install edgetpu-compiler
```

Compile:
```bash
edgetpu_compiler -s models/cylinder_detector_int8.tflite
# Output: models/cylinder_detector_int8_edgetpu.tflite
```

### Step 6 — Compare Float32 / INT8 / EdgeTPU Accuracy

```bash
/home/nam/.pyenv/versions/gesture_env/bin/python \
  scripts/compare_raw_vs_tflite.py

# Without Coral connected:
/home/nam/.pyenv/versions/gesture_env/bin/python \
  scripts/compare_raw_vs_tflite.py --no-edgetpu
```

Results saved to `results/compare_results.json`.

### Step 7 — Test Inference

```bash
# EdgeTPU (falls back to CPU automatically if Coral not connected)
/home/nam/.pyenv/versions/gesture_env/bin/python \
  scripts/test_inference.py \
  --model  models/cylinder_detector_int8_edgetpu.tflite \
  --images-dir dataset/images/
```

Saves annotated images to `outputs/inference_test/`. Only cylinder detections
(class 0) are drawn; cube predictions are suppressed.

### Step 8 — Generate Report Images

```bash
# 12 random test images, 3 columns
/home/nam/.pyenv/versions/gesture_env/bin/python \
  scripts/gen_sample_detections.py --n 12 --cols 3

# Specific images, show cube predictions too (orange)
/home/nam/.pyenv/versions/gesture_env/bin/python \
  scripts/gen_sample_detections.py \
  --images ur_2cube3cyl-24.jpg ur_1cyl1cube-36.jpg ur_1cube-23.jpg \
  --cols 3 --show-cubes

# Run on Coral EdgeTPU
/home/nam/.pyenv/versions/gesture_env/bin/python \
  scripts/gen_sample_detections.py \
  --edgetpu --model models/cylinder_detector_int8_edgetpu.tflite \
  --n 12 --cols 3
```

Colour legend: green dashed = cylinder GT · blue dashed = cube GT ·
red = cylinder prediction · orange = cube prediction (with `--show-cubes`)

## Tips

- **Dataset balance**: aim for a cylinder:cube annotation ratio below 4:1 to
  prevent the model from misclassifying cubes as cylinders.
- **Multi-object scenes**: include images with 2–3 cylinders and mixed
  cylinder+cube scenes to improve detection of all instances.
- **Threshold**: use 0.5 for production; lower to 0.3 only if false negatives
  are a concern.
- **Re-training**: collect failure cases, re-annotate in labelme, re-run
  `labelme_to_coco.py` → `train.py`.
